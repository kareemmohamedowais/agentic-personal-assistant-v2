## 🔭 نظرة عامة

مساعد شخصي ذكي متكامل يعتمد على **عدة نماذج ذكاء اصطناعي** مع قاعدة معرفة خاصة (**RAG**)، بحث ويب مباشر، ونظام شخصيات متقدم. يدعم المحادثة النصية والصوتية والصور، مع بث مباشر (Streaming) لردود الـ AI.

| المكوّن      | التقنية                            |
| ------------ | ---------------------------------- |
| Frontend     | React 18 + Vite + Tailwind CSS     |
| Backend      | Express.js (ESM) + Node.js         |
| Database     | SQLite (better-sqlite3) + WAL mode |
| Vector DB    | Pinecone (llama-text-embed-v2)     |
| AI Providers | Groq، Google Gemini، OpenRouter    |
| Auth         | JWT (7 days) + bcryptjs            |

## ✅ مميزات النظام

### 1. 🔐 نظام المصادقة (Authentication)

- **تسجيل حساب** بالاسم والبريد الإلكتروني وكلمة المرور (مشفرة بـ bcrypt)
- **تسجيل دخول** بـ JWT يصلح لـ 7 أيام
- **حماية كل الـ API routes** عبر middleware `requireAuth`
- **بيانات كل مستخدم معزولة** تماماً (Multi-tenant)

### 2. 🤖 7 موديلات ذكاء اصطناعي في Dropdown واحدة

| #   | الموديل                   | المزود     | الوصف                   |
| --- | ------------------------- | ---------- | ----------------------- |
| 1   | **Llama 3.3 70B**         | Groq       | الأقوى — الافتراضي      |
| 2   | **Kimi K2**               | Groq       | Moonshot AI — ذكاء عالي |
| 3   | **Qwen3 32B**             | Groq       | Alibaba — تفكير عميق    |
| 4   | **Llama 4 Maverick**      | Groq       | الأحدث — Llama 4        |
| 5   | **Llama 3.1 8B**          | Groq       | الأسرع — خفيف           |
| 6   | **Gemma 3 12B**           | OpenRouter | Google — مجاني تماماً   |
| 7   | **Gemini 2.5 Flash Lite** | Google     | سريع (كوتا قد تنفد)     |

- التبديل بين الموديلات **فوري** بدون إعادة تحميل
- نظام **مفضلات ★** محلّي (localStorage) — المفضلة تظهر فوق الكل
- كل مستخدم يحفظ **موديل افتراضي** خاص به في الإعدادات

### 3. ⚡ بث مباشر للردود (Real-time Streaming)

- استخدام **Server-Sent Events (SSE)** عبر `/api/chat/stream`
- الرد يظهر **حرف بحرف** أثناء التوليد
- إرسال metadata أولاً (conversationId, mediaUrl) ثم الـ tokens
- دعم **abort** — إغلاق الاتصال يوقف التوليد

### 4. 📄 نظام RAG (قاعدة المعرفة)

- **رفع مستندات** بأنواع: PDF, DOCX, PPTX, TXT, CSV
- **تقطيع ذكي** بـ `RecursiveCharacterTextSplitter` (1000 حرف مع 200 تداخل)
- **تخزين المتجهات** في Pinecone (namespace لكل مستخدم)
- **بحث تشابهي** تلقائي مع كل رسالة (أفضل **10** نتائج)
- **timeout 4 ثوانٍ** — لو البحث بطيء، يُجاب بدون context
- **حذف مستند** يحذف الـ vectors المرتبطة من Pinecone

### 5. 🎤 دعم الوسائط المتعددة (Media)

- **الصور**: JPEG, PNG, GIF, WebP — ترسل كـ base64 للموديل
- **الصوت**: MP3, WAV, WebM, OGG, MP4 — رفع ملف أو **تسجيل مباشر** عبر `MediaRecorder API` مع عداد وقت
- **تخزين الملفات** في `server/uploads/media/` مع **endpoint لعرضها** (`GET /api/media/:filename`)
- **حد أقصى 10MB** لكل ملف وسائط

### 6. 💬 إدارة المحادثات

- **إنشاء تلقائي** — أول رسالة تنشئ محادثة بعنوان ذكي
- **عرض 50 محادثة** مرتبة من الأحدث
- **حذف محادثة** يحذف كل الرسائل المرتبطة
- **history limit = 20** رسالة أخيرة ترسل مع كل طلب
- **Tags (وسوم)** — إضافة tags لكل محادثة وفلترة بها عبر `?tag=`
- **بحث في العناوين** — `?search=` في الـ API + بحث فوري في الواجهة

### 7. 📝 تلخيص تلقائي للمحادثات الطويلة

- عند تجاوز **30 رسالة** يبدأ التلخيص
- يأخذ أقدم 20 رسالة → يلخصها بالموديل الحالي → يحذفها
- **الملخص يُضاف في System Prompt** كسياق للمحادثة
- يتراكم مع الوقت (ملخصات متعددة مفصولة بـ ---)

### 8. 🎭 شخصيات الـ Agent (System Prompts)

- **5 شخصيات افتراضية** جاهزة:
  - 🤖 مساعد عام
  - 💻 مطور برمجيات
  - ✍️ كاتب محتوى
  - 🌐 مترجم
  - 📊 محلل بيانات
- **إنشاء شخصيات مخصصة** بـ اسم + وصف + system prompt + أيقونة
- **ربط شخصية بالمحادثة** — كل محادثة ممكن تستخدم شخصية مختلفة

### 9. ✨ محسّن الـ Prompts التلقائي (AI Prompt Optimizer)

- يحسّن رسالة المستخدم **قبل إرسالها للموديل**
- يستخدم **Gemini Flash Lite** كمحسّن
- يضيف وضوح، خطوات، وقيود مناسبة
- **تفعيل/تعطيل** لكل رسالة (زر ✨) أو كإعداد عام
- لا يحسّن الرسائل أقل من 5 أحرف
- يعرض للمستخدم **النسخة المحسّنة** إذا تغيرت

### 10. 🔄 نظام تدوير مفاتيح Gemini (6 Keys Rotation)

- **6 مفاتيح API** تدور تلقائياً
- تصنيف الأخطاء:
  - **RPM Limit** → حظر مؤقت (65 ثانية افتراضياً)
  - **Daily Quota** → حظر 6 ساعات
- **3 محاولات** لكل طلب (ينتقل لمفتاح آخر بعد كل فشل)
- لو كل المفاتيح محظورة → رسالة خطأ واضحة مع وقت الانتظار
- **Groq + OpenRouter**: محاولة واحدة فقط (بدون rotation)

### 11. 🔑 مفاتيح API شخصية

- المستخدم يمكنه إضافة **مفتاح Groq شخصي** و/أو **مفتاح OpenRouter شخصي**
- الأولوية: مفتاح المستخدم > مفتاح .env
- **اختبار المفتاح** قبل الحفظ عبر `/api/test-provider`
- المفاتيح **لا تُعرض كاملة** في الواجهة (فقط مؤشر "محفوظ")

### 12. 📊 لوحة الإحصائيات (Analytics)

- **عدد الرسائل الإجمالي** + **عدد المحادثات** + **عدد المستندات**
- **الرسائل اليومية** — آخر 30 يوم (للرسم البياني)
- **أنواع الملفات المرفوعة** (PDF, DOCX, ...)
- **متوسط الرسائل يومياً**
- **نسبة رسائل المستخدم مقابل الـ AI**

### 13. ⚙️ صفحة الإعدادات

- اختيار **الموديل الافتراضي** من dropdown واحدة
- تفعيل/تعطيل **محسّن الـ Prompts التلقائي**
- إدارة **مفاتيح API الشخصية** (Groq + OpenRouter)
- حالة المفاتيح مع مؤشرات بصرية

### 16. 🌙 الوضع الداكن والفاتح (Theme Toggle)

- **تبديل فوري** بين Dark/Light mode بزر في الشريط الجانبي
- الحالة محفوظة عبر **`ThemeContext`** وتُطبَّق على كل الصفحات
- أيقونة ☀️ في الوضع الداكن و 🌙 في الوضع الفاتح

### 14. 🛡️ معالجة الأخطاء (Error Handling)

- **429 Too Many Requests** → رسالة عربية واضحة مع وقت الانتظار
- **Network errors** → "تعذّر الاتصال بالخادم"
- **Server errors** → رسالة عامة ودية
- **KB search timeout** → يكمل بدون context بدل ما يفشل
- **Summarization errors** → يتم تجاهلها بصمت (non-blocking)

### 15. 🗄️ قاعدة البيانات

- **WAL mode** لأداء أفضل مع القراءة/الكتابة المتزامنة
- **Migrations تلقائية** — أعمدة جديدة تُضاف بدون كسر
- **6 جداول**: users, conversations, messages, documents, prompts, user_settings
- **Foreign keys + CASCADE delete** — حذف مستخدم يحذف كل بياناته

## 🆕 الميزات المُضافة حديثاً

الميزات التالية تم تطويرها وإضافتها مؤخراً كجزء من تحسين النظام:

### 1. 🌐 بحث الويب المباشر (Web Search)

- بحث مباشر في الإنترنت عبر **Tavily REST API**
- يبحث تلقائياً عندما يحتاج الموديل لمعلومات حديثة
- يُرجع أفضل 5 نتائج مع عناوين وروابط وملخصات
- يعمل كـ **Tool** داخل نظام الـ Agent — الموديل يقرر متى يستخدمه
- Fallback ذكي: لو البحث فشل، يكمل الرد بمعرفته الداخلية

### 2. ✏️ إعادة تسمية المحادثات (Rename Conversations)

- **زر ✏️ بجانب كل محادثة** في الشريط الجانبي
- **تحرير inline** — يتحوّل العنوان لـ `<input>` مباشرة
- حفظ بـ **Enter** أو إلغاء بـ **Escape**
- يحدّث العنوان فوراً محلياً + في الـ DB عبر `PUT /api/conversations/:id/rename`

### 3. 📌 تثبيت المحادثات (Pin Conversations)

- زر **تثبيت/إلغاء تثبيت** لكل محادثة
- المحادثات المثبتة تظهر **فوق الكل** دائماً في الشريط الجانبي
- مع مؤشر بصري واضح للمحادثات المثبتة

### 4. 🛡️ لوحة تحكم Admin

- صفحة **admin محمية** للمستخدم ذو الصلاحية `role: "admin"`
- **إحصائيات شاملة**: إجمالي المستخدمين، الرسائل، الرسائل اليومية
- **قائمة المستخدمين** مع عدد رسائل كل واحد وتاريخ التسجيل
- **تفعيل/تعطيل** حسابات المستخدمين
- **تغيير الأدوار** (user ↔ admin)
- معالجة أخطاء 403 مع **زر إعادة تسجيل دخول** عند انتهاء صلاحية الـ JWT

### 5. 🎭 نظام الشخصيات المتقدم (Persona System)

- **5 شخصيات احترافية** مع System Prompts مفصّلة:
  - 🤖 **مساعد عام** — يبحث في KB + ويب، منظّم ومختصر
  - 💻 **مطور برمجيات** — code blocks بالـ language tag، تحليل bugs، clean code
  - ✍️ **كاتب محتوى** — يتأكد من النبرة، hook-body-CTA structure
  - 🌐 **مترجم** — دقة تقنية + طبيعية إبداعية، يسأل عن اللغة المستهدفة
  - 📊 **محلل بيانات** — منهجية 6 نقاط: توضيح → ملخص → insights → جداول → charts → limitations
- **ربط الشخصية بالمحادثة** — كل محادثة تحفظ شخصيتها في الـ DB
- **استعادة الشخصية تلقائياً** عند فتح محادثة قديمة
- **PromptPicker دائم** فوق مربع الكتابة — مع tooltip يعرض وصف كل شخصية
- **Badge في الـ Header** يعرض الشخصية النشطة
- **Upsert ذكي** — الـ DB تُحدَّث تلقائياً عند إعادة تشغيل السيرفر

### 6. 🖥️ معاينة HTML بشاشة كاملة (Full-screen Preview)

- زر **"معاينة"** في كل code block يحتوي HTML
- يفتح **modal بشاشة كاملة** (full-screen overlay) بدل iframe صغير
- **header بطراز المتصفح** (نقاط حمراء/صفراء/خضراء + زر إغلاق)
- إغلاق بـ **Escape** أو **النقر خارج النافذة**
- iframe معزول بـ `allow-scripts allow-same-origin`

### 7. 💎 بطاقة رد AI احترافية (AI Message Card)

- تصميم **Card احترافي** بدل bubble بسيط:
  - **خط تمييز علوي** بـ gradient (indigo → purple)
  - **Header**: أيقونة gradient + "مساعد الذكاء الاصطناعي" + badge الشخصية + الوقت النسبي
  - **مؤشر "يكتب..."** متحرّك أثناء الـ streaming
  - **Markdown محسّن**: headings منسّقة، bullets ملوّنة، blockquote بحدود، روابط بلون indigo
  - **Footer تفاعلي** (يظهر عند hover): زر نسخ (✓ تأكيد) + إعجاب 👍 + عدم إعجاب 👎
- **رسائل المستخدم**: bubble مبسّط + وقت نسبي + avatar
- **وقت نسبي ذكي**: "الآن" → "منذ 5 د" → "منذ 2 س" → "6 مارس"

### 8. 🔎 بحث في المحادثات (Sidebar Search)

- **حقل بحث** في أعلى الشريط الجانبي
- بحث فوري في **عناوين المحادثات**
- تصفية النتائج أثناء الكتابة

### 9. 🏷️ نظام Tags للمحادثات

- إضافة **وسوم ملونة** لكل محادثة
- **فلترة المحادثات** بـ tag عبر `?tag=` في الـ API
- Tags تظهر كـ **badges صغيرة** تحت عنوان المحادثة في الشريط الجانبي

### 10. 🔍 تبديل بحث الويب في الواجهة (Web Search Toggle)

- **زر تفعيل/تعطيل** بحث الويب في مربع الكتابة
- يرسل `enableWebSearch: true/false` مع كل رسالة
- المستخدم يتحكم متى يريد البحث الخارجي

### 11. ✨ نقطة نهاية تحسين الـ Prompts (Optimize Endpoint)

- **`POST /api/optimize-prompt`** — endpoint منفصل لتحسين الـ prompt قبل الإرسال
- يستخدم Gemini Flash Lite كمحسّن
- يُعيد النسخة المحسّنة فقط إذا اختلفت عن الأصل

## 💡 10 اقتراحات لتقوية المشروع

### 1. 🔍 Web Search Tool — بحث في الإنترنت

**الوضع الحالي:** الموديل يعتمد فقط على معرفته + المستندات المرفوعة.
**الإضافة:** دمج أداة بحث ويب (مثل Tavily, SerpAPI, أو Brave Search) بحيث لو المستخدم سأل عن شيء حديث أو يحتاج معلومة من الإنترنت، يبحث تلقائياً ويرجع النتائج كـ context.
**التأثير:** يحوّل المساعد من chatbot محدود المعرفة إلى assistant يعرف كل شيء حديث.

### 2. 🧠 Memory طويلة المدى (Long-term Memory)

**الوضع الحالي:** التلخيص يحفظ سياق المحادثة فقط.
**الإضافة:** نظام memory يحفظ **حقائق عن المستخدم** عبر المحادثات (اسمه، تخصصه، تفضيلاته، مشاريعه).  
مثلاً: "المستخدم مبرمج Python يعمل على مشروع React" — ويحقنها تلقائياً في كل محادثة جديدة.
**التأثير:** المساعد يبقى "يعرفك" حتى لو بدأت محادثة جديدة.

### 3. 🛠️ Function Calling / Tool Use (Agent حقيقي)

**الوضع الحالي:** الموديل يولّد نص فقط.
**الإضافة:** إعطاء الموديل أدوات يقدر ينفذها:

- **حاسبة** — عمليات رياضية دقيقة
- **تنفيذ كود** — Python sandbox
- **إنشاء ملفات** — PDF, Excel
- **قراءة URL** — يفتح رابط ويلخصه
  **التأثير:** يتحول من "chatbot" إلى "agent" حقيقي يقدر يعمل tasks.

### 4. 📱 تحويل لـ PWA (Progressive Web App)

**الوضع الحالي:** يعمل في المتصفح فقط.
**الإضافة:** إضافة manifest.json + Service Worker + أيقونات → يتحول لتطبيق يتثبت على الموبايل والديسكتوب.
**التأثير:** تجربة استخدام كأنه تطبيق native + يعمل offline (عرض المحادثات المحفوظة) + push notifications.

### 5. 🔗 مشاركة المحادثات (Share Conversations)

**الوضع الحالي:** المحادثات خاصة بالمستخدم فقط.
**الإضافة:** زر "مشاركة" ينشئ رابط عام (read-only) يمكن إرساله لأي شخص. يمكن إضافة خيار "مشاركة مع انتهاء صلاحية" أو "محادثة عامة/خاصة".
**التأثير:** مفيد جداً للتعاون ومشاركة حلول أو أبحاث.

### 6. 🌓 تخصيص الواجهة المتقدم _(Theme Toggle مُنفَّذ بالفعل)_

**الوضع الحالي:** Dark/Light mode ✅ مُنجز. ما ينقص:
**الإضافة:** خيارات ألوان accent قابلة للتخصيص (indigo/blue/green/pink) + حجم الخط + تغيير كثافة الخلفية.
**التأثير:** شخصنة كاملة للواجهة — كل مستخدم يختار مظهره المفضل.

### 7. 📤 تصدير المحادثات (Export)

**الوضع الحالي:** المحادثات محفوظة في الـ DB فقط.
**الإضافة:** أزرار تصدير بصيغ:

- **Markdown** (.md)
- **PDF** (مع formatting)
- **JSON** (للنسخ الاحتياطي)
- نسخ الرد الواحد بضغطة زر

### 8. 🔀 Fallback تلقائي بين الموديلات

**الوضع الحالي:** لو الموديل المختار فشل (429, error) → يرجع error للمستخدم.
**الإضافة:** نظام fallback chain — لو Gemini فشل → يحاول Groq Llama 3.3 → لو فشل → يحاول OpenRouter. المستخدم يحدد ترتيب الأولوية.
**التأثير:** المساعد **ما بيقف أبداً** — دايماً في موديل شغال يرد.

### 9. 🏷️ تنظيم المحادثات (Folders) _(Tags و Search مُنفَّذان بالفعل)_

**الوضع الحالي:** Tags ✅، Search ✅، Pin ✅ كلها مُنجزة. ما ينقص:
**الإضافة:** **فولدرات/مجلدات** — تنظيم المحادثات في مجموعات (عمل، دراسة، شخصي) مع drag-and-drop.

### 10. 📈 Rate Limiting + Usage Quotas

  **التأثير:** حماية من الاستخدام المفرط + إمكانية تحويل المشروع لـ SaaS.

### 11. 🗣️ Text-to-Speech — تحويل الردود لصوت

**التأثير:** تجربة استخدام مختلفة تماماً — المساعد يتحدث إليك. مفيد أثناء القيادة أو عند عدم إمكانية القراءة.

### 14. 🔔 نظام الإشعارات الذكية (Smart Notifications)

### 16. 🧪 نظام تقييم جودة الإجابات (LLM Evaluation Pipeline)

### 19. 🚦 Queue + Background Workers للمهام الثقيلة

- مهام ingest والتلخيص والمعالجة الطويلة تعمل في الخلفية
- إعادة محاولة تلقائية عند الفشل
- مراقبة حالة المهام (pending, processing, failed, done)
- عزل الضغط: API تظل سريعة حتى مع ملفات كبيرة

**التأثير:** ثبات أعلى بكثير تحت الضغط وعدد مستخدمين أكبر بدون انهيار الأداء.

### 20. 🌍 دعم تعدد اللغات الكامل (i18n)

**الإضافة:** نظام i18n احترافي (مثل `i18next`):

**التأثير:** يفتح المنتج لسوق أوسع فورًا ويعطي انطباع احترافي جدًا للمستخدمين الدوليين.

### 21. 🤝 محادثات تعاونية (Collaborative Chat)

### 22. 📷 قراءة النصوص من الصور (OCR)

- يعيد النص المستخرج قابلاً للنسخ والتحرير
- يدعم العربية والإنجليزية معاً
- النص المستخرج يمكن إرساله مباشرة كرسالة للمحلل

**التأثير:** صورة فاتورة / جدول بيانات / شاشة كود → نص قابل للتحليل في ثوانٍ.

### 23. 🔍 بحث دلالي في المحادثات (Semantic Search)

- "ابحث عن كل مرة تحدثت عن الـ API" → يجد 5 محادثات

**التأثير:** ذاكرة قابلة للبحث — المستخدم لا يحتاج يتذكر أين تحدّث عن شيء معين.

**التأثير:** يحوّل المساعد من مجرد أداة بحث إلى **رفيق تعلم فعّال** — مفيد للطلاب والمتخصصين.

## 📊 أفكار مخصصة لتحليل البيانات

أفكار متقدمة لتحويل شخصية **محلل البيانات** من chatbot عادي إلى أداة تحليل حقيقية:

### 1. 📤 رفع ملفات CSV/Excel مباشرة في المحادثة

**الفكرة:** السماح للمستخدم بإرفاق ملف بيانات (CSV, Excel, JSON) مباشرة في المحادثة عند استخدام شخصية محلل البيانات.

**التنفيذ:**

- Backend يقرأ الملف بـ `xlsx` أو `csv-parse` ويستخرج البيانات
- يرسل أول 50 صف + metadata (أسماء الأعمدة، الأنواع، عدد الصفوف) كـ context للموديل
- الموديل يستطيع طلب صفوف إضافية أو فلترات عبر tool calling

**القيمة:** المستخدم يقول "حلل هذا الملف" ويرفقه مباشرة — بدون نسخ/لصق.

---

### 2. 📈 توليد رسوم بيانية تفاعلية (Charts)

**الفكرة:** عندما يطلب المستخدم رسماً بيانياً، المحلل يولّد كود **Chart.js** أو **Recharts** يُعرض مباشرة في المحادثة.

**التنفيذ:**

- الموديل يولّد JSON بتنسيق محدد: `{ type: "bar", labels: [...], datasets: [...] }`
- Frontend يكتشف هذا التنسيق ويرسم Chart تفاعلي (hover, zoom, download)
- يدعم: Bar, Line, Pie, Scatter, Heatmap
- زر **"تحميل كصورة PNG"** لكل رسم

**القيمة:** الرسوم البيانية تنطق — بدل ما يقرأ أرقام جافة، يشوف الأنماط بصرياً.

---

### 3. 🧮 حاسبة إحصائية مدمجة (Statistical Calculator Tool)

**الفكرة:** إضافة tool يستدعيها الموديل لإجراء حسابات إحصائية دقيقة بدل التخمين.

**التنفيذ:**

- Tool: `calculate_statistics({ data: [...], operations: ["mean", "median", "std", "correlation"] })`
- يستخدم مكتبة `simple-statistics` أو `mathjs` على السيرفر
- يدعم: المتوسط، الوسيط، الانحراف المعياري، الارتباط، الانحدار الخطي، t-test, chi-square
- النتائج تُرجع بتنسيق جدول مع تفسير بالعربية

**القيمة:** LLMs تخطئ في الحسابات — هذا يضمن **دقة 100%** في الأرقام.

---

### 4. 🔄 تحويل البيانات (Data Transformation Pipeline)

**الفكرة:** الموديل يستطيع تنظيف وتحويل البيانات عبر tools مخصصة.

**العمليات المدعومة:**

- **تنظيف**: إزالة القيم الناقصة، حذف المكرر، تصحيح الأنواع
- **تحويل**: pivot/unpivot, group by + aggregate, merge جداول
- **إنشاء أعمدة**: حسابية (عمود A × عمود B) أو مشتقة (السنة من التاريخ)
- **فلترة**: شروط مركّبة (المبيعات > 1000 AND المدينة = "القاهرة")

**القيمة:** المستخدم يصف ما يريد بالعربية والنظام ينفّذ — "اجمع المبيعات حسب المنطقة".

---

### 5. 📝 توليد تقارير PDF تلقائية

**الفكرة:** بعد التحليل، زر **"إنشاء تقرير"** يولّد PDF احترافي.

**محتوى التقرير:**

- عنوان + تاريخ + اسم المحلل
- ملخص تنفيذي (Executive Summary)
- الرسوم البيانية المولّدة في المحادثة
- الجداول الإحصائية
- التوصيات والاستنتاجات
- ملحق: المنهجية والبيانات الخام

**التنفيذ:** استخدام `pdfkit` أو `puppeteer` لتحويل HTML مُنسّق لـ PDF.

**القيمة:** من محادثة عادية إلى **تقرير رسمي** بضغطة زر — يوفّر ساعات عمل.

---

### 6. 🗄️ اتصال مباشر بقواعد البيانات (SQL Connector)

**الفكرة:** المستخدم يضيف connection string لقاعدة بياناته، والمحلل يكتب SQL ويستعلم مباشرة.

**التنفيذ:**

- دعم: SQLite, PostgreSQL, MySQL
- الموديل **يولّد SQL** → النظام يراجعه (read-only فقط) → ينفذه → يعرض النتائج
- **حماية**: فقط SELECT مسموح، مع timeout 10 ثوانٍ، وحد 1000 صف
- المستخدم يقول: "كم عدد الطلبات هذا الشهر؟" → الموديل يكتب SQL → ينفذه → يعرض الجواب

**القيمة:** يتحول من محلل "نظري" إلى أداة تحليل **حقيقية** متصلة ببيانات فعلية.

---

### 7. 🕐 تحليل السلاسل الزمنية (Time Series Analysis)

**الفكرة:** أدوات متخصصة لتحليل البيانات عبر الزمن.

**القدرات:**

- كشف **الاتجاهات** (Trend Detection) — هل المبيعات ترتفع أم تنخفض؟
- كشف **الموسمية** (Seasonality) — هل هناك أنماط شهرية/أسبوعية؟
- كشف **القيم الشاذة** (Anomaly Detection) — يوم مبيعات أعلى بـ 3x من المعتاد
- **تنبؤ بسيط** (Forecasting) — توقع القيم القادمة باستخدام المتوسط المتحرك أو الانحدار
- رسم بياني خطي مع تظليل مناطق الثقة

**القيمة:** أي شخص عنده بيانات شهرية (مبيعات، زيارات، إيرادات) يحتاج هذا التحليل.

---

### 8. 🎯 Dashboard تفاعلي لكل محادثة تحليل

**الفكرة:** عند تحليل بيانات، يُنشأ تلقائياً **dashboard مصغّر** فوق المحادثة.

**المكوّنات:**

- **KPI Cards**: أرقام رئيسية (إجمالي المبيعات, المتوسط, النمو %)
- **رسوم بيانية مصغّرة** (sparklines)
- **فلتر ديناميكي** — المستخدم يغيّر الفترة أو الفئة والـ dashboard يتحدث
- يُحفظ مع المحادثة ويمكن مشاركته

**القيمة:** يحوّل المحادثة من نص جامد إلى **أداة تحليل تفاعلية مرئية**.

---

### 9. 🔁 مقارنة الفترات الزمنية (Period Comparison)

**الفكرة:** أمر بسيط مثل: "قارن مبيعات Q1 مع Q2" والنظام يعمل مقارنة شاملة.

**المخرجات:**

- جدول مقارنة side-by-side
- نسبة التغيير لكل مؤشر (↑ 15% أو ↓ 8%)
- رسم بياني مقارن (bars جنب بعض)
- ملخص نصي: "المبيعات ارتفعت 15% لكن الأرباح انخفضت 3% بسبب..."
- إبراز **أكبر تغيير إيجابي** و**أكبر تغيير سلبي**

**القيمة:** المقارنة الزمنية هي **أكثر تحليل طلباً** في بيئات العمل.

---

### 10. 🧠 تحليل نصي (Text Analytics / NLP)

**الفكرة:** توسيع المحلل ليشمل تحليل **البيانات النصية** وليس فقط الرقمية.

**القدرات:**

- **تحليل المشاعر** (Sentiment Analysis) — إيجابي/سلبي/محايد لتعليقات العملاء
- **استخراج الكلمات المفتاحية** — أكثر الكلمات تكراراً + word cloud
- **تصنيف تلقائي** — تصنيف التذاكر/الشكاوى حسب الموضوع
- **تلخيص الاستبيانات** — 1000 رد مفتوح → 5 نقاط رئيسية
- يعمل بالعربية والإنجليزية

**القيمة:** كثير من البيانات المهمة **نصية** (تقييمات، شكاوى، ردود استبيانات) — هذا يفتح باب تحليل جديد بالكامل.

---

### 11. 📋 قوالب تحليل جاهزة (Analysis Templates)

**الفكرة:** قوالب مُعدّة مسبقاً يختار منها المستخدم بضغطة واحدة.

**القوالب المقترحة:**

| القالب                | الوصف                                             |
| --------------------- | ------------------------------------------------- |
| 📊 تحليل مبيعات       | إيرادات, أفضل المنتجات, الموسمية, التنبؤ          |
| 👥 تحليل عملاء        | التوزيع الديموغرافي, الاحتفاظ, القيمة الدائمة     |
| 💰 تحليل مالي         | P&L, التدفق النقدي, النسب المالية, المقارنة       |
| 📱 تحليل تطبيق        | DAU/MAU, معدل الاحتفاظ, funnel, مصادر الزيارات    |
| 📣 تحليل حملة تسويقية | ROI, تكلفة الاكتساب, معدل التحويل                 |
| 📋 تحليل استبيان      | التوزيعات, المتوسطات, الترابطات, النتائج المفتوحة |

كل قالب يسأل أسئلة محددة ثم يبدأ تحليل منهجي.

**القيمة:** يخفّض **حاجز الاستخدام** — حتى غير المتخصصين يحللون بياناتهم باحتراف.

---

### 12. ⚠️ تنبيهات ذكية على البيانات (Data Alerts)

**الفكرة:** تحديد عتبات (thresholds) والنظام ينبّهك عند تجاوزها.

**أمثلة:**

- "نبّهني إذا انخفضت المبيعات اليومية عن 5000"
- "أعلمني إذا ارتفع معدل الإرجاع فوق 10%"
- يتحقق دورياً (يومياً مثلاً) ويرسل إشعار
- يظهر كرسالة تلقائية في المحادثة أو كـ push notification

**القيمة:** المحلل لا ينتظر سؤالك — **يخبرك بالمشاكل قبل ما تكتشفها**.
